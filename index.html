CloStat

Github Repository

Technologies used: Firebase Realtime Database, Node.js, npm, handlebars, jQuery, OpenWeatherMap API

CloStat provides users with clothing recommendations based on the up-to-date weather information. It is a virtual closet that let the user keeps track of the status of clothing (clean or worn) to allow a more accurate recommendation. It is created using Firebase Realtime Database, Node.js, npm, handlebars, OpenWeatherMap API.

BrainStorm

In my Human-Computer Interaction (HCI) Programming Studio class (Cogs 121), my group brainstormed ideads about what mobile web application to make that centered around the theme of interacting with real world data. We all liked the idea of having something to narrow down on what clothes we should wear from day-to-day because California's ever-changing weather and temperature made it hard for us to wear something we like while still enjoying the outdoor.

Paper Prototype

My group used paper prototype to design what would later be called Clostat. We mapped out each of the screens that will be displayed when the users click different buttons on the web app. In additon, we also mapped out the different ways the user might interact with the app and think of ways to get around any user problems.

The Application

For this project, I implemented user authentication using Firebase Authentication, implemented the OpenWeatherMap API, designed and implemented Firebase Realtime Database, and helped designed the application.

Some challenges I faced while working on this project was implementing Firebase Realtime Database as my whole group has never used it before. I also tried to optimize the design of the database to decrease the time the application need to retrieve data. This was my second time developing in web, so there were still a lot of things that I knew very little about, such as node.js, express, and npm. I did some research to get some general ideas of them and their basic usage.

Markit

Github Repository
Development Plan

Technologies used: Node.js, npm, handlebars, jQuery, Google Analytics, Quagga API, bootstrap

Markit allow users to scan barcodes to compare the user reviews on store products to make a wiser purchase. Markit is designed to combat the struggle of struggling to find the best quality product from the massive amount of choices offered to the consumer in store. It brings the online ecommerce feature of reviewing products to the physical world.

BrainStorm

In my HCI Interaction Design class (Cogs 120), my group brainstormed ideas about what mobile web application we can make that deals with everyday problems and satisfy the theme of Environmental Interaction, and we all thought of the problem we had while grocery shopping. As we walked down the aisle, the sheer amount of choices overwhelmed us. We thought of the different ways we shopped and decided to integrate the online shopping experience with real world grocery shopping.

Paper Prototype

We first used paper prototype to mapped out the most essential features of the app.

Next, we used InVision to improve our paper prototype design and made it an interactive app to further test how users would interact when using the app. Last, we built the mobile web app MarkIt using HTML, CSS, JavaScript, Express, Node.js, and Quagga API.

The Application

For this project, I set up version control, created the HTML skeleton for the application, delegated tasks to group memebers, implemented the Quagga API to detect and scan barcdoes, implemented Google Analytics to perform A/B testing, and implemented search feature to search for items.

Some challenges I faced while working on this project was trying to find a good API that can detect and scan barcodes easily. I also had to do some research into barcodes as I was not aware of the different types of barcodes out there. It was also my first time doing web development, so I knew next to nothing about HTML, CSS, and Javascript, much less web frameworks and libraries. I had to learn the basics of HTML, CSS, and Javascript on my own, while delegating tasks to my group members, who also had no experience in web development.

WhereYouAt(WYA)

Github Repository

WYA helps people coordinate while trying to meet up for an event. Like other event apps, it allows the user to make an event and invite others. What sets it apart from other event app is that it lets people see where the other event attendees are at if they chose to show their location. This feature enables attendees to guide one another to the event in the case that they get lost or canno find one another.

TritonEd Redesign

Design Documentation

In my Human-Computer Interaction class, my group redesigned the mobile version of TritonEd, UC San Diego's Learning Management System. After the first round of user testing to uncover the usability errors, we chose to focus on the navigation process in the homepage to fix several of the usability errors. We came up with four redesigns on paper, then use InVision and Adobe XD to create two final redesigns. In our second round of user testing, users prefer our redesign because of its simplicity and ease of use. We tried to introduce several new features or ways of interaction, but it deviated too much from the original website. Users either were unaware of these new features or were confused by them. Overall, we learned that despite how unoptimized the original site is for mobile, students adopted to the design and got around the design flaws.
